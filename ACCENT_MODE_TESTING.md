# Чек-лист проверки: режим «Акцент на показатель»

Сборка и запуск в симуляторе:
```bash
SDK=$(ls -d ~/Library/Application\ Support/Garmin/ConnectIQ/Sdks/connectiq-sdk-mac-9.1.0-* | head -1)
"$SDK/bin/monkeyc" -f monkey_clear.jungle -o bin/SOFBlck_fenix8solar47mm.prg \
  -d fenix8solar47mm -y /Users/dim/garmin/clock/pilot/developer_key.der
monkeydo bin/SOFBlck_fenix8solar47mm.prg fenix8solar47mm
```

---

## 1. Базовая навигация по показателям
- [ ] **DOWN ×1** из выключенного режима → акцент на «заряд дней» (idx 0), значение ×2.
- [ ] **DOWN ×1** ещё раз → idx 1 «пульс» ×2, idx 0 вернулся к базовому шрифту.
- [ ] **UP ×1** → шаг против часовой стрелки; шрифт/частота предыдущего сброшены.
- [ ] Полный круг **DOWN ×12** → возврат к idx 0 (цикл `0→1→…→11→0`).
- [ ] Полный круг **UP ×12** → возврат к idx 0 в обратную сторону.

## 2. Выход из режима (кнопка BACK)
- [ ] **BACK ×1** в активном режиме → всё к базовому виду, режим выключен (НЕ уход на эко-экран).
- [ ] **BACK ×1** вне режима → эко-экран (прежнее поведение).
- [ ] **BACK ×2** (в режиме и вне) → меню навигации (не сломано).

## 3. Двойные жесты не сломаны
- [ ] **DOWN ×2** → экран таймеров (`TimerView`).
- [ ] **UP ×2** → список будильников (`AlarmListView`).
- [ ] **START ×1** → переключение GPS; **START ×2** → список секундомеров.
- [ ] Задержка ~0.5 с на одиночное переключение акцента — ожидаема (плата за двойные жесты).

## 4. Отрисовка ×2 (визуально)
- [ ] Крупное значение НЕ вылезает за край экрана.
- [ ] Чёрная подложка перекрывает соседние метки (а не перекрыта ими).
- [ ] Значение сдвинуто ближе к центру (`R_accent = s(90)`), ориентация по кругу как у обычных меток.
- [ ] Проверить на **260 px** (`fenix8solar47mm`) и **454 px** (`fenix847mm`).
- [ ] При необходимости подобрать `R_accent`/паддинг в `TactixView.drawAccentText()`.

## 5. Live-датчики
- [ ] idx 1 «пульс» — обновляется ~1 с (живой Sensor HR), а не «замерзает».
- [ ] При уходе с idx 1 пульсометр гаснет (экономия батареи).
- [ ] idx 2 «высота» — свежее значение (~5 с); при отсутствии позиции → fallback на историю.
- [ ] idx 8 «давление» — live (sea-level); при `null` (нет GPS) → fallback на ambient-историю.
- [ ] Нюанс: при переходе live↔fallback значение может скакнуть — это принято.

## 6. Совместимость с компасом/пеленгом (главный риск рефактора)
- [ ] Включить компас → стрелки N/S работают как раньше (heading приходит).
- [ ] Включить пеленг на метку → стрелка и дистанция корректны.
- [ ] Компас активен + выбрать акцент idx 1 → heading И пульс приходят одновременно.
- [ ] Компас активен + выбрать idx 2/8 → heading И высота/давление приходят.
- [ ] `accentOff` (BACK) при активном компасе → heading остаётся, HR/барометр гаснут (если не нужны компасу).
- [ ] Прогрев магнитометра ~2 с после включения — ошибка «NO COMPASS» не мигает ложно.

## 7. Жизненный цикл
- [ ] Уход на эко-экран (BACK вне режима) и возврат → сенсоры восстановлены, акцент сброшен.
- [ ] Перезапуск приложения → акцент выключен по умолчанию (`accentIndex = -1`).

## Затронутые файлы
| Файл | Изменения |
|------|-----------|
| `source/TactixApp.mc` | состояние акцента, `_syncSensors()`, `onSensor()`, методы `accentNext/Prev/Off` |
| `source/TactixDelegate.mc` | одиночные UP/DOWN, выход по BACK |
| `source/TactixView.mc` | `placeMetric()` + частотный кэш, `drawAccentText()` (×2, подложка) |

## Известные риски (см. PLAN_ACCENT_MODE.md → «Риски»)
1. Регрессия компаса/пеленга из-за единого Sensor-диспетчера.
2. HR-сенсор может конфликтовать с heading на реальном устройстве.
3. ×2-bbox за краем экрана — подбор `R_accent`.
4. Live-давление зависит от GPS-фикса.
