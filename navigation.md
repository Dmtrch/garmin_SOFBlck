## 0. Обзор и архитектура

Текущее поведение: двойное `BACK` на главном экране (`TactixDelegate.onBack()`) вызывает `app.toggleCompass()` — включает/выключает компас.

Новое поведение: двойное `BACK` открывает экран **«Меню навигации»** (`NavMenu`) с тремя пунктами:
1. **Компас вкл/выкл** — toggle + возврат на главный экран.
2. **Установить метку** → подменю работы с одной активной меткой (см. раздел 4).
3. **Показать направление на метку** → выбор метки → возврат на главный, отрисовка стрелки-пеленга и расстояния в метрах.

Новые модули (под `source/`):
- `NavManager.mc` — модель данных и персистентность меток + активного режима пеленга.
- `NavMenuDelegate.mc` — корневое меню навигации.
- `WaypointMenuDelegate.mc` — подменю «Установить метку».
- `WaypointListDelegate.mc` — список меток для удаления и для выбора пеленга.
- `WaypointEditView.mc` + `WaypointEditDelegate.mc` — ручной ввод широты/долготы.
- `MapPickView.mc` + `MapPickDelegate.mc` — выбор точки на офлайн-карте устройства.

Изменения в существующих модулях:
- `TactixDelegate.mc` — заменить `toggleCompass()` на `pushView(NavMenu)`.
- `TactixApp.mc` — добавить состояние `bearingActive`, `bearingTargetIdx`, кэш `Position.Info`; продолжать держать состояние компаса.
- `TactixView.mc` — отрисовка стрелки пеленга по периметру + текста расстояния.
- `manifest_clear.xml` — добавить `iq:uses-permission id="Positioning"` (уже есть) и проверить `Storage`/`Background` при необходимости.
- `resources/strings/strings.xml` — добавить строки локализации (RU/EN через `System.LANGUAGE_RUS`, паттерн как в `AlarmListDelegate`).

## 2. NavManager — модель и хранилище

Хранение в `Application.Storage` под ключом `nav_waypoints` (массив словарей).

Структура одной метки:
```
{
  "name" => String,   // короткое имя, авто "WP1", "WP2", ...
  "lat"  => Double,   // градусы, -90..90
  "lon"  => Double,   // градусы, -180..180
  "t"    => Long      // unix timestamp создания
}
```

- [x] **2.2** Реализовать `NavManager.load() as Array` — читает Storage, возвращает массив (может быть пустой).
- [x] **2.3** Реализовать `NavManager.save(arr as Array) as Void`.
- [x] **2.4** Реализовать `NavManager.add(lat, lon) as Boolean` — добавляет метку с автоименем, возвращает `false` если уже `MAX` меток.
